We have a new CLI tool to make syncing WordPress sites between S3 environments much easier and safer. Instead of manually running AWS CLI commands, you can now use a simple, validated command with built-in safety features.

Installation

Just run this one command:
npm install -g wfuwp

Prerequisites:

  • Node.js (most of you already have this)
  • AWS CLI installed and configured with your WFU credentials

How to Use

The main command is syncs3 and it follows this pattern:
wfuwp syncs3 [options]

Environments: dev, uat, pprd, prod

Examples

  • Basic sync with confirmation prompt
    wfuwp syncs3 43 uat pprd
  • Preview what would be synced (dry run)
    wfuwp syncs3 43 uat pprd --dry-run
  • Show detailed output with all file transfers
    wfuwp syncs3 43 uat pprd --verbose
  • Skip confirmation (useful for scripts)
    wfuwp syncs3 43 uat pprd --force
  • Combine flags
    wfuwp syncs3 43 uat pprd --verbose --dry-run

Safety Features

  • Input validation - Won’t let you use invalid site IDs or environments
  • Confirmation prompts - Asks before making changes (unless you use --force)
  • Dry run mode - Preview changes with --dry-run before running for real
  • AWS CLI checks - Verifies your AWS setup before attempting sync

Getting Help

wfuwp --help # General help
wfuwp syncs3 --help # Specific command help
wfuwp --version # Check version

Troubleshooting

If you get AWS-related errors, make sure your AWS CLI is configured:
aws configure list # Check your AWS setup

If your AWS keys have expired, follow this guide to refresh them.

The tool uses the same S3 buckets and permissions you already have - it’s just a safer, easier way to run the sync commands.
